## Creating a Subscription

`create_subscritpion_service(org_id, user, db)` at `backend/services/org_service.py:77` looks up the organization filtered by both `org_id` and `owner_id == user_id` — if not found, raises `HTTPException(404, "Organization not found or you are not the owner")` (lines 80-86). If the org's `organization_plan` is already `"PRO"`, raises `HTTPException(400, "Organization is already on the Pro plan")` (lines 88-89). Reads `STRIPE_PRO_PRICE_ID` from the environment — if missing, raises `HTTPException(500, "Billing is not configured")` (lines 91-93). Creates a Stripe checkout session via `stripe.checkout.Session.create` with `payment_method_types=["card"]`, `mode="subscription"`, a single `line_items` entry with the PRO price and quantity 1 (lines 95-101). The `success_url` is `f"{os.getenv('FRONTEND_URL', 'http://localhost:3000')}/success?org_id={org_id}&session_id={{CHECKOUT_SESSION_ID}}"` and the `cancel_url` is `f"{os.getenv('FRONTEND_URL', 'http://localhost:3000')}/cancel"` (lines 102-103). Both `metadata` and `subscription_data.metadata` carry `{"org_id": str(org_id), "user_id": str(user_id)}` (lines 104-105). `client_reference_id` is set to `str(org_id)` (line 106). Returns `session.url` (line 109).

## Confirming an Upgrade

`confirm_upgrade_service(org_id, session_id, user, db)` at line 112 logs the invocation with org and session IDs (line 113). Looks up the organization by owner — if not found, raises `HTTPException(404, "Organization not found")` (lines 116-122). Checks for an existing `Organization_payments` with `status == "active"` for the org — if the org is already `"PRO"` with an active subscription and a `stripe_subscription_id`, returns `{"status": "active", "plan": org.organization_plan}` immediately (lines 124-130). As a fallback for local development (where the webhook may not have arrived yet), it attempts to verify the checkout session directly with Stripe (lines 134-176). Calls `stripe.checkout.Session.retrieve(session_id)` (line 136). Extracts `org_id` from session metadata's `org_id` or `client_reference_id` (line 144). If the resolved org ID doesn't match the requested `org_id`, logs a warning and returns `{"status": "pending"}` (lines 150-152). If `payment_status != "paid"` or `mode != "subscription"`, returns pending (lines 154-155). Retrieves the subscription via `stripe.Subscription.retrieve`, checks the price against `STRIPE_PRO_PRICE_ID` (lines 161-167). Calls `_activate_pro_for_org(db, org_id, subscription_id, price_id)`, refreshes the org, returns `{"status": "active", "plan": org.organization_plan}` (lines 169-171). On any exception, logs and returns pending (lines 172-174). Returns pending by default if no `session_id` (line 176).

## Stripe Webhook Handler

`handle_stripe_webhook_service(payload, sig_header, db)` at line 241 first checks for `STRIPE_WEBHOOK_SECRET` in the environment — if missing, logs an error and raises `HTTPException(500, "Webhook not configured")` (lines 242-245). If no signature header is provided, raises `HTTPException(400, "Missing Stripe signature")` (lines 247-248). Constructs a Stripe event via `stripe.Webhook.construct_event(payload, sig_header, webhook_secret)` — a `ValueError` raises `HTTPException(400, "Invalid payload")` and a `stripe.error.SignatureVerificationError` raises `HTTPException(400, "Invalid signature")` (lines 250-255). Extracts `event_type`, `data_object`, and `pro_price_id` (lines 257-259). Logs the received event (line 261).

For `"checkout.session.completed"` (lines 263-298): ignores sessions where `mode != "subscription"` or `payment_status != "paid"` (lines 264-268). Extracts `org_id` from metadata's `org_id` or `client_reference_id` (lines 270-279). Retrieves the subscription via `stripe.Subscription.retrieve` — a StripeError raises `HTTPException(502, "Stripe error")` (lines 285-289). Verifies the price matches `STRIPE_PRO_PRICE_ID` (lines 291-295). Calls `_activate_pro_for_org` (line 297).

For `"customer.subscription.updated"` and `"customer.subscription.created"` (lines 300-320): extracts `subscription_id`, `status`, and `price_id` (lines 301-304). If the price doesn't match PRO, returns early (lines 306-307). If `status` is `"active"` or `"trialing"`, resolves `org_id` from the subscription metadata (falling back to the existing payment row's `organization_id`) and calls `_activate_pro_for_org` (lines 309-317). If `status` is `"canceled"`, `"unpaid"`, `"past_due"`, or `"incomplete_expired"`, calls `_deactivate_pro_for_subscription(db, subscription_id, status)` (lines 318-319).

For `"customer.subscription.deleted"` (lines 322-325): calls `_deactivate_pro_for_subscription(db, subscription_id, "cancelled")`.

The catch-all at line 327 returns `{"received": True}` for unhandled event types.

## Cancelling a Subscription

`cancel_subscription_service(org_id, user, db)` at line 330 looks up the organization by owner — if not found, raises `HTTPException(404, "Organization not found or you are not the owner")` (lines 333-339). If the plan is not `"PRO"`, raises `HTTPException(400, "Organization does not have an active Pro plan")` (lines 341-342). Looks up the active payment for the org — if none exists or it has no `stripe_subscription_id`, raises `HTTPException(404, "No active subscription found for this organization")` (lines 344-350). Calls `stripe.Subscription.cancel(payment.stripe_subscription_id)` — on `StripeError`, raises `HTTPException(502, f"Stripe error: {str(e)}")` (lines 352-355). Sets `payment.status = "cancelled"`, `org.organization_plan = "FREE"`, and commits (lines 357-359). Calls `create_log` with `action="subscription_cancelled"` (line 361). Returns `{"message": "Subscription cancelled. Your organization has been downgraded to the Free plan."}` (line 363).
