# Glossary

The glossary is an alphabetical list of technical, specific, or complex terms accompanied by short definitions.

**Purpose:** Helps readers understand technical or specialized terms that may not be familiar to everyone.

| Term | Definition |
|---|---|
| REST API | A programming interface based on the HTTP protocol that allows applications to communicate with each other through standardized requests. |
| Container | A lightweight and portable unit for running applications, isolated from the host system. |
| CRUD | The four basic data operations: Create, Read, Update, and Delete. |
| JWT | A signed digital token used to authenticate a user and securely transmit their information between client and server. |
| Latency | The delay between sending a request and receiving the response. |
| LLM | An artificial intelligence model trained on large amounts of text, capable of understanding and generating natural language. |
| ORM | A programming technique that maps objects in a language to tables in a relational database. |
| RAG | An approach that combines information retrieval from a knowledge base with a language model to generate context-aware responses. |
| WebSocket | A communication protocol providing real-time, two-way exchanges between a client and a server over a persistent connection. |
| Webhook | A mechanism by which a service automatically sends data to an external URL when an event occurs. |
